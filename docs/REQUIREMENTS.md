# 운영 전 요구사항

## 현재 MVP 범위

현재 버전은 다음 목적에 적합합니다.

- 고객관리 화면 시연
- 영업 프로세스 검증
- 교육용 바이브코딩 실습
- 내부 아이디어 제안
- 사용자 피드백 수집

## 실제 운영을 위한 필수 요구사항

### 인증과 권한

- 사내 계정 로그인
- 관리자·영업담당자·조회자 권한 구분
- 퇴직자와 부서 이동자 권한 회수
- 세션 만료와 비밀번호 정책

### 데이터

- 중앙 고객 데이터베이스
- 고객 중복 검증
- 데이터 백업과 복구
- 수정 이력과 삭제 정책
- 엑셀·CRM 데이터 가져오기

### 보안

- HTTPS
- 개인정보 암호화
- 연락처와 이메일 마스킹
- 다운로드 권한 통제
- 접속 및 변경 로그
- 외부 공개 저장소에 실제 고객정보 저장 금지

### 업무 연동

- Microsoft Outlook 일정
- 이메일 발송 이력
- Teams 미팅
- 견적서와 제안서 파일 관리
- ERP 또는 기존 CRM 연동

### 추천 개발 단계

1. 현재 MVP 사용자 테스트
2. 필수 필드와 화면 흐름 확정
3. 데이터베이스 설계
4. 로그인과 권한 구현
5. API 및 업무시스템 연동
6. 보안 검토
7. 파일럿 운영
8. 정식 배포
